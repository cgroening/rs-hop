# Security Policy

## Supported versions

Security fixes are applied to the latest released version on
[crates.io](https://crates.io/crates/rs-hop). Please make sure you are on the
newest release before reporting an issue.

| Version | Supported          |
| ------- | ------------------ |
| 0.2.x   | :white_check_mark: |
| < 0.2   | :x:                |

## Reporting a vulnerability

Please report suspected vulnerabilities privately rather than opening a public
issue. Use GitHub's
[private vulnerability reporting](https://github.com/cgroening/rs-hop/security/advisories/new)
for the repository, or contact the maintainer directly through the account on
the [project page](https://github.com/cgroening/rs-hop).

Include enough detail to reproduce the problem (hop version, operating system,
and the steps or configuration that trigger it). You can expect an
acknowledgement within a few days, and a fix or mitigation plan once the report
is confirmed.
